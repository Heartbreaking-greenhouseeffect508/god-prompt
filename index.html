What It Is

PHASE 0

Auto-Classification

Detects 9 task types — build, debug, refactor, content, design, ship, analyze, automate, plan — and routes to the right protocol automatically.

IRON LAWS

Three Non-Negotiables

Understand before acting. Scope is sacred. Evidence before claims. These laws apply to every task, every time, without exception.

THE GATE

Verification Gates

No "should work now." Every claim requires proof — run the check, read the output, confirm the result. Mandatory on every task, no exceptions.

How It Works

PHASE 0 CLASSIFY — auto-detect task type and route
PHASE 1 UNDERSTAND — read, trace, explore. ask if ambiguous
PHASE 2 PLAN — state changes, order, success criteria
PHASE 3 EXECUTE — one change at a time. surgical. conservative
PHASE 4 VERIFY — run proof. read output. then claim result
PHASE 5 DELIVER — report changes, verification, rollback plan

Every task flows through all six phases. Depth scales with complexity — a typo fix breezes through in seconds, a new feature goes deep.

FAQ

What is GodPrompt?

A 1,145-line behavioral framework that replaces 30+ individual AI skills with one unified system. It auto-detects task types and applies production-grade engineering discipline — understand, plan, execute, verify, deliver — to every task automatically.

How is this different from a regular system prompt?

Regular system prompts set tone and constraints. GodPrompt implements an entire engineering methodology — a 6-phase protocol with auto-classification, iron laws, verification gates, and anti-pattern detection. It changes how the AI works, not just how it talks.

Which AI models does this work with?

Optimized for Claude Code as a native skill. The single-file GodPrompt.md also works with Claude Projects, ChatGPT Custom Instructions, or any AI that accepts system-level instructions.

Yes. MIT licensed, open source, free to use and modify. Contributions welcome.
